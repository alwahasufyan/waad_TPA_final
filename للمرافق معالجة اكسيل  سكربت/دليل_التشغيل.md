# دليل التشغيل السريع — مُطابِق قوائم الأسعار لنظام TPA الطبي

تقرير تسليم مختصر لمهندس يشغّل الأداة على جهاز جديد.

---

## 1) ما هي الأداة؟
تأخذ قائمة أسعار **فوضوية** واردة من مستشفى/مختبر (Excel / PDF / PowerPoint / CSV)،
تطابقها مع **ملف المرجع الرسمي** للخدمات، وتُنتج ملف Excel **مرتّبًا جاهزًا للرفع**
على النظام، مع تعبئة الكود والتصنيف الرئيسي والفرعي تلقائيًا، وفصل ما يحتاج مراجعة بشرية.

---

## 2) محتويات المجلد (ما الذي تستلمه)
| الملف | الوصف | ضروري؟ |
|------|--------|:---:|
| `tpa_service_mapper.py` | السكربت الرئيسي (المطابقة + الإخراج) | ✅ |
| `ingest.py` | محرك قراءة الصيغ (Excel/PDF/PPTX/CSV) — **يجب أن يكون بجانب السكربت** | ✅ |
| `قائمة التصنيفات المعتمدة.xlsx` | **قائمة التصنيفات المعتمدة** — المصدر الوحيد لتصنيفات المخرجات | ✅ |
| `process_all.py` | معالجة دفعية: كل ملفات «تحتاج تصنيف» → «جاهز» بأمر واحد | اختياري || `process_old_root.py` | معالجة الملفات القديمة في جذر «جاهز» من مصادر «تحتاج تصنيف قديمة» | اختياري || `collect_approved.py` | أداة تجميع الصفوف المعتمدة بعد المراجعة | اختياري |
| `medical_synonyms.json` | قاموس المرادفات الطبية (وسّعه مع الوقت) | اختياري |
| `Price_List_Contract_1_Output.xlsx` | **ملف المرجع** (الكتالوج الرسمي) — 6 أعمدة | ✅ |
| `README.md` | الشرح التفصيلي الكامل | مرجعي |
| `جاهز\` | مخرجات القوائم المكتملة (منظمة بمجلدات حسب النوع) | — |

### التصنيفات المعتمدة ⭐ (جديد)
- كل خدمة في المخرجات تُسند **حصرًا** إلى أحد تصنيفات
  `قائمة التصنيفات المعتمدة.xlsx` (الرمز + الاسم + التصنيف الأب إيواء/عيادات خارجية).
- أُضيف تصنيفان من تصدير نظام وعد (`medical-categories-*.xlsx`):
  **CAT030 نظارة طبية** و**CAT031 علاج الأسنان (تركيب-تقويم-زراعة)** —
  موجودان حاليًا في `قائمة التصنيفات المعتمدة_محدثة.xlsx` (يفضّلها السكربت
  تلقائيًا إن وُجدت بجانب الأصلية).
- ملف المرجع «عقد 1» يستخدم ترقيمًا قديمًا؛ يحوّله السكربت تلقائيًا إلى
  الأكواد المعتمدة (مثلًا: القديم CAT025 «أدوية روتينية» ← المعتمد CAT024).
- يظهر سطر تحقق في نهاية كل تشغيل: «✓ كل الصفوف مرتبطة بتصنيفات معتمدة».
- لتغيير ملف التصنيفات: `--categories "مسار_الملف.xlsx"`.

### معيار الجاهزية = صحة التصنيف ⭐ (جديد)
التصنيف (الرئيسي ثم الفرعي) هو ما يحدد سياسة المنافع في النظام، والكود
يُولَّد تلقائيًا والسعر من قائمة المرفق. لذلك تذهب الخدمة إلى ورقة
«النتيجة» (جاهزة للرفع) إذا كان تصنيفها **موثوقًا** من أي مصدر:
1. مطابقة سعرية بمرجع العقد، أو
2. تصنيف موثق من نظام أودو السابق، أو
3. نوع المرفق (أسنان/بصريات — عبر `--hint`).

ولا يبقى في «تحتاج مراجعة» إلا ما صُنّف تقديريًا بالقواعد الطبية أو
بالافتراضي. عمود «سبب النتيجة» يبيّن مصدر كل تصنيف، والملخص يفصّل:
مطابقة عقد / تصنيف موثق / مراجعة.

### قاعدة معرفة نظام أودو السابق ⭐ (جديد)
- `build_odoo_kb.py` يبني `odoo_knowledge.json` (~9,900 اسم خدمة بتصنيف
  موثق) من تصديري أودو: «متغير المنتج (product.product)» و«الخدمات الصحية
  (product.template)»، محوّلًا فئات أودو وعلاماته إلى الأكواد المعتمدة.
- السكربت الرئيسي يحمّلها تلقائيًا إن وُجدت بجانبه: أي خدمة في «تحتاج
  مراجعة» يوجد اسمها في النظام السابق تأخذ تصنيفه الموثق (بدل تخمين
  الكلمات المفتاحية) مع ملاحظة «التصنيف من نظام أودو السابق».
- أولوية التصنيف: تلميح المجموعة ← مرجع العقد ← قاعدة أودو ← القواعد
  الطبية ← الافتراضي. أعد تشغيل `build_odoo_kb.py` عند تحديث ملفات أودو.

### تلميح نوع المرفق `--hint` ⭐ (جديد)
| التلميح | الأثر |
|---------|-------|
| `dental` | كل الخدمات أسنان بعيادات خارجية: تركيبات/تقويم/زراعة ← CAT031، والباقي ← CAT027 |
| `optics` | النظارات والعدسات ← CAT030، والكشوف ← CAT028، والافتراضي CAT028 |
| `physio` | الافتراضي للخدمات غير المصنفة ← CAT026 العلاج الطبيعي المقرر |

`process_all.py` يطبّق التلميح تلقائيًا حسب اسم مجلد المجموعة
(اسنان/بصريات/علاج طبيعي).

### المعالجة الدفعية (جديد)
```powershell
.venv\Scripts\python.exe -X utf8 process_all.py
# مجموعة واحدة فقط:
.venv\Scripts\python.exe -X utf8 process_all.py --only "مختبرات"
```
تعالج كل ملفات «تحتاج تصنيف» وتضع النتائج في «جاهز\<المجموعة>\نتيجة_*.xlsx».

### الدفعة القديمة (جذر «جاهز») ⭐
الملفات القديمة مباشرة في جذر `جاهز\` (مثل `نتيجة_ابن_سينا.xlsx`) مصادرها
في مجلد `تحتاج تصنيف قديمة\` (PDF/PPTX/Excel). لإعادة توليدها بالتنسيق الجديد:
```powershell
.venv\Scripts\python.exe -X utf8 process_old_root.py
```
يربط السكربت كل مصدر بمخرجه (جدول `JOBS` داخله) ويكتب النتيجة في جذر `جاهز\`.

> ملاحظة: تحتاج مصادر PDF/PowerPoint المكتبتين `pdfplumber` و`python-pptx`
> (مدرجتان في المتطلبات بالقسم 3).

> ضع `ingest.py` في **نفس مجلد** `tpa_service_mapper.py` وإلا لن يعمل السكربت.

## 3) المتطلبات (تثبيت لمرة واحدة)
يحتاج **Python 3.9+** والحِزم التالية:
```bash
pip install pandas openpyxl pdfplumber python-pptx rapidfuzz
```
- `rapidfuzz` تُحسّن الدقة والسرعة (يعمل بدونها بمحرك داخلي بديل).
- `pdfplumber` لقراءة PDF و`python-pptx` لقراءة PowerPoint.

### إن لم يكن Python مثبّتًا (بديل عبر uv — مُجرّب على هذا الجهاز)
```powershell
# تثبيت uv ثم إنشاء بيئة معزولة وتثبيت الحِزم
uv venv --python 3.12
uv pip install pandas openpyxl pdfplumber python-pptx rapidfuzz
# بعدها شغّل بـ: .venv\Scripts\python.exe بدل python
```

## 4) طريقة التشغيل
```bash
python tpa_service_mapper.py \
    --reference "Price_List_Contract_1_Output.xlsx" \
    --input     "قائمة_المستشفى.pdf" \
    --output    "النتيجة.xlsx" \
    --synonyms  "medical_synonyms.json" \
    --threshold 85
```

### المعاملات
| المعامل | إلزامي | الوصف |
|---------|:---:|--------|
| `--reference` | ✅ | ملف المرجع (الكتالوج الرسمي) |
| `--input` | ✅ | القائمة الواردة — يقبل: `.xlsx .xls .csv .pdf .pptx` |
| `--output` | ✅ | اسم ملف الإخراج المرتّب |
| `--synonyms` | ❌ | قاموس المرادفات JSON (يرفع الدقة) |
| `--threshold` | ❌ | حد قبول المطابقة 0–100 (افتراضي **85**) |
| `--code-prefix` | ❌ | بادئة الكود المولّد للخدمات بلا كود (افتراضي `NEW`) |

### مثال على Windows / PowerShell (بيئة uv)
```powershell
.venv\Scripts\python.exe tpa_service_mapper.py `
  --reference "Price_List_Contract_1_Output.xlsx" `
  --input "مستشفى بيروت.xlsx" `
  --output "نتيجة_بيروت.xlsx" `
  --synonyms "medical_synonyms.json" --threshold 85
```

## 5) ملف المخرجات — ورقة بيانات واحدة ⭐ (تنسيق جديد)
1. **الخدمات Services** — كل الخدمات في جدول واحد:
   - الأعمدة الستة الأولى (الخضراء) مطابقة تمامًا لملف الرفع.
   - الصفوف البيضاء: تصنيف **موثق** (مرجع العقد / أودو / نوع المرفق) — جاهزة.
   - الصفوف **الحمراء في نهاية الجدول**: تصنيف تقديري — مهمة المراجع الوحيدة:
     تأكيد/تصحيح التصنيف الرئيسي والفرعي من **القائمة المنسدلة** في كل خلية
     (القوائم مقصورة على التصنيفات المعتمدة حصرًا — لا كتابة يدوية).
   - عمود «حالة التصنيف»: ✔ موثق / ⚠ راجِع التصنيف، وعمود «سبب النتيجة»
     يوضح مصدر كل تصنيف.
2. **ملخص Summary** — إحصائيات (مطابقة عقد / تصنيف موثق / مراجعة).
3. **تعليمات الاستخدام** — شرح داخل الملف.
4. **قوائم التصنيفات** (مخفية) — مصدر القوائم المنسدلة.

المراجعة أصبحت تصحيح تصنيف مباشرًا في مكانه عبر القوائم المنسدلة المعتمدة.

### إنتاج ملف الرفع النهائي (بعد المراجعة)
```bash
python collect_approved.py "النتيجة.xlsx"
```
يُنتج «..._للاعتماد_النهائي.xlsx» بالأعمدة الستة فقط جاهزًا للّصق —
أو ببساطة انسخ الأعمدة الستة الأولى من ورقة «الخدمات» مباشرة.

---

## 6) ضبط الدقة مقابل التغطية
- الافتراضي **85** متوازن. الدرجات **85–88** قد تحوي حالات حدّية (نظرة بشرية سريعة).
- دقة أعلى: `--threshold 90` | تغطية أوسع: `--threshold 80`.
- لا تُفرط في خفض العتبة (تحمي من أخطاء خطيرة مثل EMG مقابل ECG).

---

## 7) تحسين النتائج تدريجيًا
كلما ظهرت خدمة في «تحتاج مراجعة» وكان يجب مطابقتها، أضِف صيغتها إلى
`medical_synonyms.json` تحت الاسم المعتمد بالمرجع:
```json
"complete blood count": ["cbc", "صورة دم كاملة", "تعداد الدم"]
```

## 8) ملاحظات تشغيلية مهمّة
- **ingest.py مطلوب بجانب السكربت** (يستورده داخليًا).
- يقرأ Excel **بكل أوراقه**، ويكتشف الأعمدة بأي ترتيب وحتى بلا عناوين.
- يُصحّح **اتجاه النص العربي المعكوس** في PDF تلقائيًا؛ قد تبقى بقايا ترميز بسيطة
  (مثل لامات/أرقام معكوسة) تُعالَج في المراجعة البشرية.
- PowerPoint: إن كانت البيانات في مربّعات نصّية منفصلة، يزاوج السعر بأقرب اسم
  موضعيًا (تقريبي — يُراجَع).
- تفاوت نِسب المطابقة طبيعي: المرجع كتالوج عيادات خارجية، فقوائم التحاليل تحقّق
  أعلى نسبة، والعمليات/الإيواء تذهب بحق إلى «تحتاج مراجعة».
- لمعالجة عدة ملفات دفعة واحدة، كرّر أمر التشغيل بملف إدخال/إخراج مختلف لكل قائمة.
