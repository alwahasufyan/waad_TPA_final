# مُرتِّب ومُطابِق قوائم الأسعار لنظام TPA — النسخة 2

أداة تأخذ قائمة أسعار **فوضوية** واردة من مستشفى/صيدلية/مختبر (بأي صيغة:
Excel أو PDF أو PowerPoint أو CSV)، وتطابقها مع **ملف المرجع الرسمي**
للخدمات، ثم تُنتج ملف Excel **مرتّبًا وجاهزًا للاستيراد** داخل نظام التأمين،
مع تعبئة الكود والتصنيف الرئيسي والفرعي تلقائيًا، وفصل ما يحتاج مراجعة بشرية.

---

## 1) المتطلبات (مرة واحدة)
```bash
pip install pandas openpyxl pdfplumber python-pptx rapidfuzz
```
- `rapidfuzz` اختيارية لكنها تُنصح (أدق وأسرع). بدونها يعمل السكربت تلقائيًا
  بمحرك **فهرس كلمات داخلي** سريع لا يحتاج أي تثبيت.
- `pdfplumber` لقراءة PDF و`python-pptx` لقراءة PowerPoint.

## 2) الملفات
| الملف | الوصف |
|------|--------|
| `tpa_service_mapper.py` | السكربت الرئيسي (المطابقة + الإخراج) |
| `ingest.py` | محرك قراءة الصيغ (Excel/PDF/PowerPoint/CSV) — **مطلوب بجانب السكربت** |
| `process_all.py` | معالجة دفعية: كل ملفات مجلد «تحتاج تصنيف» → «جاهز\<المجموعة>» |
| `process_old_root.py` | معالجة دفعية للملفات القديمة في جذر «جاهز» من مصادرها في «تحتاج تصنيف قديمة» |
| `collect_approved.py` | تجميع الصفوف بعد المراجعة من ورقة «الخدمات» إلى ملف رفع نهائي |
| `build_odoo_kb.py` | يبني `odoo_knowledge.json` من تصديري أودو (قاعدة تصنيف موثقة) |
| `medical_synonyms.json` | قاموس المرادفات الطبية (وسّعه مع الوقت) |
| `قائمة التصنيفات المعتمدة.xlsx` | **المصدر الوحيد للتصنيفات** (الرمز + الاسم + الأب إيواء/عيادات خارجية) |
| `Price_List_Contract_1_Output.xlsx` | ملف المرجع الرسمي (الكتالوج) |

> ضع `ingest.py` في نفس مجلد `tpa_service_mapper.py`.

### المعالجة الدفعية وبنية المجلدات
- **`تحتاج تصنيف\<المجموعة>\`** (اسنان/بصريات/علاج طبيعي/مختبرات/مصحات): مصادر
  القوائم الجديدة. شغّل `python process_all.py` فتخرج النتائج إلى
  **`جاهز\<المجموعة>\نتيجة_*.xlsx`**. (`--only "مختبرات"` لمجموعة واحدة).
- **`تحتاج تصنيف قديمة\`**: مصادر دفعة أقدم (PDF/PPTX/Excel). سكربت
  `process_old_root.py` يربط كل مصدر بمخرجه ويضعه في **جذر `جاهز\نتيجة_*.xlsx`**.
- كل المخرجات (الدفعتان) بالتنسيق الموحد الجديد نفسه (القسم 5).

## 3) التشغيل
```bash
python tpa_service_mapper.py \
    --reference Price_List_Contract_1_Output.xlsx \
    --input     "قائمة_مستشفى.pdf" \
    --output    "النتيجة.xlsx" \
    --synonyms  medical_synonyms.json \
    --threshold 85
```
`--input` يقبل: `.xlsx .xls .csv .pdf .pptx` — والباقي كما هو لأي صيغة.

## 4) الصيغ المدعومة
- **Excel**: يقرأ **كل الأوراق**، يكتشف صف العناوين وأعمدة (الاسم/السعر/الكود)
  بأي ترتيب، ويتجاوز الصفوف/الأعمدة الفارغة وعناوين الأقسام.
- **PDF**: يستخرج الجداول من كل الصفحات (pdfplumber)، **ويصحّح تلقائيًا اتجاه
  النص العربي المعكوس** الذي يظهر في كثير من ملفات PDF (يكشف الانعكاس ولا يلمس
  النص السليم).
- **PowerPoint**: يقرأ الجداول؛ وإن كانت البيانات في مربّعات نصّية منفصلة
  (شائع)، يزاوج كل سعر بأقرب اسم حسب موضعه على الشريحة (تقريبي — يُراجَع).
- **CSV**: مفصول بأي فاصل.

اكتشاف الأعمدة يعمل **حتى بلا عناوين** عبر تصنيف كل عمود حسب خصائصه
(نسبة الأرقام، الكسور، تنوّع القيم، طول النص، نمط الأكواد) واستبعاد عمود ترقيم
الصفوف تلقائيًا.

## 5) ملف المخرجات — ورقة «الخدمات» الموحدة (التنسيق الحالي) ⭐
كل ملف مخرج يحوي **4 أوراق فقط**:
1. **الخدمات Services** — جدول واحد يجمع كل الخدمات:
   - **الأعمدة الستة الأولى** (بالأخضر) مطابقة تمامًا لأعمدة ملف الرفع:
     `service_name، service_code، contract_price، main_category،
     sub_category، notes` — انسخها كما هي بعد المراجعة.
   - الصفوف **الموثقة التصنيف أولًا** (خلفية بيضاء)، ثم صفوف **«راجِع
     التصنيف» حمراء في نهاية الجدول** (تصنيف تقديري يحتاج تأكيدًا).
   - عمودا **التصنيف الرئيسي (D)** و**الفرعي (E)** فيهما **قائمة منسدلة**
     على كل خلية مقصورة على التصنيفات المعتمدة حصرًا (لا كتابة يدوية).
   - عمود **«حالة التصنيف»**: ✔ موثق / ⚠ راجِع التصنيف، وعمود **«سبب
     النتيجة»** يوضح مصدر كل تصنيف (عقد / أودو / نوع المرفق / تقديري).
   - فلتر تلقائي على الجدول للفرز والترشيح.
2. **ملخص Summary** — إحصائيات (مطابقة عقد / تصنيف موثق / مراجعة).
3. **تعليمات الاستخدام** — شرح داخل الملف.
4. **قوائم التصنيفات** (مخفية) — مصدر القوائم المنسدلة.

> **سير العمل للمراجعة:** افتح ورقة «الخدمات» → الصفوف الحمراء فقط هي مهمة
> المراجع: أكّد أو صحّح التصنيف الرئيسي والفرعي من القائمة المنسدلة → انسخ
> الأعمدة الستة الأولى إلى ملف الرفع (أو شغّل `collect_approved.py`).

## 6) كيف تعمل المطابقة (بالترتيب)
1. تطابق تام بعد التنظيف. 2. المرادفات. 3. توحيد عربي/إنجليزي وحذف كلمات
الوصف العامة (serum/plasma/عينة…). 4. تقريبي (تشابه مجموعات + احتواء + تسلسل).

## 7) ضبط الدقة مقابل التغطية (مهم)
- الافتراضي **85** متوازن. المطابقات **85–88** قد تحوي حالات حدّية — نظرة بشرية
  سريعة على أدنى الدرجات. المطابقات **≥90** آمنة جدًا.
- دقة أعلى: `--threshold 90` | تغطية أوسع: `--threshold 80`.
- العتبة تحمي عمدًا من أخطاء خطيرة (مثل EMG مقابل ECG). لا تُفرط في خفضها.

## 7.5) إنتاج ملف الرفع النهائي (بعد المراجعة)
بعد تأكيد/تصحيح التصنيف في الصفوف الحمراء بورقة «الخدمات»، أنتج ملف الرفع بإحدى طريقتين:
- **(أ) نسخ مباشر:** ظلّل الأعمدة الستة الأولى كاملة من ورقة «الخدمات» والصقها في ملف الـ price_list المعتمد.
- **(ب) أداة التجميع:** احفظ الملف ثم شغّل:
  ```bash
  python collect_approved.py "نتيجة_الخدمات.xlsx"
  ```
  تُنتج ملفًا واحدًا «..._للاعتماد_النهائي.xlsx» بالأعمدة الستة فقط، جاهزًا للّصق مباشرة
  (تدعم الورقة الموحدة الجديدة وتظل متوافقة مع أوراق التنسيق القديم).


كلما ظهرت خدمة تحتاج مطابقة ولم تُطابَق، أضِف صيغتها إلى
`medical_synonyms.json` تحت الاسم المعتمد في المرجع — بلا تعديل للكود:
```json
"صورة دم كاملة": ["cbc", "complete blood count", "تعداد الدم"]
```

## 7.6) الأكواد (service_code) ⭐
- تحتفظ الأداة بكود المستشفى الأصلي من الملف الوارد (مثل ACC0000045، G-A-T-03).
- للخدمات المطابَقة تُستخدم أكواد المرجع المعتمدة.
- أي خدمة **بلا كود إطلاقًا** تحصل على كود مولّد في **عمود الكود** بصيغة
  `NEW00001` (تُغيّر البادئة بـ `--code-prefix`). فلا يبقى صف بلا كود.
- التصنيف الفرعي (CAT0xx) يبقى في عموده، والكود في عموده — لا خلط بينهما.
- التصنيف الرئيسي دائمًا «إيواء» أو «عيادات خارجية» فقط؛ التفصيل في الفرعي.

## 8.5) التصنيف الطبي التلقائي (جديد) ⭐
- **مطابقة ثنائية اللغة**: إن كان للملف عمودان (عربي + إنجليزي/لاتيني)، تطابق
  الأداة الاسمين معًا وتأخذ الأفضل، وتعرضهما معًا (`CBC | صورة دم كاملة`).
- **التصنيفات المعتمدة (المصدر الوحيد)**: تُحمَّل من
  `قائمة التصنيفات المعتمدة.xlsx` (بديل `--categories`)، وكل صف في المخرجات
  يرتبط حصرًا بأحد تصنيفاتها (الرمز + الاسم + الأب إيواء/عيادات خارجية).
  أكواد المرجع القديمة تُحوَّل تلقائيًا للأكواد المعتمدة، ويُطبع سطر تحقق
  في نهاية كل تشغيل.
- **مُصنِّف طبي**: يُسند **لكل خدمة** تصنيفًا رئيسيًا وفرعيًا تقديريًا بقواعد طبية (تحاليل،
  CT/MRI، أسنان، عيون، علاج طبيعي، أجهزة، أدوية، جراحة، تخدير، ولادة وقيصرية،
  عناية مركزة...) — **حتى لو لم تُطابَق سعريًا**. فتخرج صفوف «تحتاج مراجعة»
  مصنّفة وجاهزة، ويبقى تأكيد الكود فقط.
- **حارس الأجسام المضادة**: ترفض الأداة تلقائيًا خلط فئات IgG/IgM/IgA/IgE
  (مثل HBc IgG مع HBc IgM) لمنع أخطاء فوترة خطيرة.

> التصنيف التلقائي لا يرفع **نسبة المطابقة** (المطابقة بالاسم تعتمد على وجود
> الخدمة في المرجع)، لكنه يرفع **الجاهزية**: كل خدمة تخرج مصنّفة ومنظّمة.

## 9) نتائج الاختبار على ملفاتك الحقيقية (مرجع: عقد 1)
| المصدر | الصيغة | إجمالي | مطابَقة | نسبة |
|--------|--------|--------|---------|------|
| الصفوة (تحاليل) | PDF | 260 | 128 | 49% |
| بيروت | Excel | 5089 | 496 | 10% |
| ابن سينا | Excel | 1477 | 150 | 10% |
| الاستشاري | Excel | 972 | 90 | 9% |
| الحكمة | PowerPoint | 97 | 11 | 11% |

**لماذا النِّسب متفاوتة؟** ليست عيبًا في الأداة. المرجع «عقد 1» كتالوج
عيادات خارجية بالأساس، بينما هذه كتالوجات مستشفيات كاملة فيها آلاف العمليات
الجراحية وغرف الإيواء غير الموجودة في العقد، فتذهب بحق إلى Need Review. ولهذا
قائمة التحاليل (الصفوة) حقّقت أعلى نسبة. رافعتا الرفع: (1) مرجع أشمل،
(2) إثراء المرادفات. **دقة المطابَق عالية** (مثل: ANTI GLIDIN IgG ←
ANTI-GLIADIN IgG، CA15.3 ← CA 15-3، 25(OH)VITAMIN D ← VIT D).
