# نموذج مسارات النسخ الاحتياطي في WAAD

## القرار المعتمد

النسخ الاحتياطية الحالية في BKP-2 تُحفظ داخل السيرفر فقط، عبر مسارات معتمدة ومركّبة داخل Docker.

جهاز مستخدم المتصفح ليس وجهة نسخ احتياطي. إذا أراد المدير نسخة على جهازه الشخصي، يستخدم زر التحميل من سجل النسخ الاحتياطية.

## التطوير المحلي

- مسار الجهاز المضيف، لأغراض التطوير فقط: `BACKUP_LOCAL_1_HOST`
- القيمة الافتراضية في المثال: `D:/waad_backups`
- مسار Docker الداخلي المعتمد: `/app/backups/local1`

الـ backend يكتب فقط إلى المسار الداخلي:

```text
/app/backups/local1
```

## الإنتاج

- مسار السيرفر المضيف: `/opt/waad/backups/local`
- مسار Docker الداخلي: `/app/backups/local1`

الإعدادات المقترحة في `.env.production`:

```text
BACKUP_LOCAL_1_HOST=/opt/waad/backups/local
BACKUP_LOCAL_1_CONTAINER=/app/backups/local1
```

## ما لا تفعله الواجهة

- لا تسمح للمستخدم بكتابة مسار مثل `C:\Users\...`.
- لا تحفظ تلقائيًا إلى Desktop أو أي مسار على جهاز مستخدم المتصفح.
- لا تنفذ restore إنتاجي.
- لا تنفذ reset خطير.

## ما يظهر في الواجهة

تعرض الواجهة:

- اسم الوجهة.
- نوع الوجهة: مسار محلي على السيرفر.
- مسار السيرفر المعتمد.
- مسار Docker الداخلي.
- حالة قابلية الكتابة.
- مدة الاحتفاظ.

الحقول القابلة للتعديل حاليًا:

- التفعيل.
- اسم الوجهة.
- مدة الاحتفاظ.

المسارات read-only وتُدار من Docker أو ملف البيئة.

## طبقات الحماية المستقبلية

هذه المرحلة لا تنفذ BKP-3. لاحقًا:

1. Acronis من Libyan Spider لحماية السيرفر/VPS بالكامل.
2. Offsite/Object Storage لتجنب امتلاء السيرفر بالملفات والصور والنسخ.
3. S3-compatible storage مثل Cloudflare R2 أو Wasabi أو SFTP.
