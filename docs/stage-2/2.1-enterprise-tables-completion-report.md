## 1. Analyze (findings)

The frontend has three "shared" table components, and they are very unevenly used:

| Component | Usage | Note |
|---|---|---|
| **`UnifiedMedicalTable`** | **20 pages** | The real system-wide table ("THE SINGLE SOURCE OF TRUTH FOR ALL TABLES") |
| `GenericDataTable` | 3 pages (settlement) | Feature-rich (tanstack), niche |
| `UnifiedDataTable` | **0 pages** | **Dead code** |

`UnifiedMedicalTable` was already strong: sticky header, sorting, pagination, loading/empty/error states, row selection, row expansion, export, filters, search, refresh. The genuine gap versus the UI/UX Constitution ("*Tables should remember: Filters, Sorting, Visible Columns, Page Size, User Preferences*") was **personalization / preference memory** — no density control and no column show/hide.

Constraint discovered: pagination and sorting are **controlled by the parent pages** (each of the 20 pages owns that state). So page-size persistence would require touching 20 pages — out of scope and risky. **Density and column-visibility, however, can live entirely inside the component** — the safe, high-leverage win.

## 2. Implement

**Enhanced the dominant component `UnifiedMedicalTable`, opt-in and fully backward-compatible:**

- Added an optional **`persistKey`** prop. When provided, a small right-aligned toolbar appears above the table with:
  - **Density toggle** (comfortable ↔ compact) — overrides the MUI table `size`, remembered in `localStorage` (`waad:tbl:<key>:density`).
  - **Column show/hide menu** — checkbox list of columns; hidden columns are removed from header + body; remembered in `localStorage` (`waad:tbl:<key>:hiddenCols`). Guard: the **last visible column can never be hidden**.
- **Backward compatibility (critical, since 20 pages use this component):** when `persistKey` is absent, `prefsEnabled=false` → **no toolbar**, `resolvedDensity = size`, `hiddenColumnIds = []` → `visibleColumns === normalizedColumns`. The 17 non-opt-in pages render **byte-for-byte as before**. Only the 3 pages below opt in.
- Routed the three internal column usages (colSpan, header map, body map) through `visibleColumns` (equals the full set when nothing is hidden).
- Persistence is `try/catch`-guarded (safe if `localStorage` is unavailable).

**Enabled `persistKey` on three core, high-traffic list screens** (demonstrate + let you test):
- `pages/members/UnifiedMembersList.jsx` → `persistKey="members-list"`
- `pages/providers/ProvidersList.jsx` → `persistKey="providers-list"`
- `pages/employers/EmployersList.jsx` → `persistKey="employers-list"`

**Cleanup:** removed the confirmed-dead `UnifiedDataTable` (0 usages) and its barrel re-export — one fewer parallel table, supporting the "cleaner / more consistent" goal.

**No workflow change. No business-rule change. No redesign.** The tables show the same data, in the same order, gated by the same permissions — users simply gain optional density and column control that persists.
