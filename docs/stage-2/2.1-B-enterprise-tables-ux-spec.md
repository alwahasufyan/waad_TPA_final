# Stage 2.1-B — Enterprise Tables UX — Spec & Sub-Increment Plan

**Origin:** Your additional observations after 2.1. Captured in full so nothing is lost.
**Overriding rule (yours):** every improvement must be **opt-in / Backward Compatible** and must **not** change business workflow, break the system, or slow it down. Desktop-first, all browsers, scalable.

---

## Full Requirements (captured)

1. **Search/Filters → collapsible Side Filter Panel** — filters move to an openable/closable side panel; the table stays the largest element; remember open/closed state per user.
2. **Default ordering = Newest First** — every table shows newest data first by default, with the ability to change sort and a clear current-sort indicator.
3. **Table space is king** — any layout gives the table maximum space.
4. **Unify ALL tables** — Members, Employers, Providers, Claims, Payments, Settlements, Reports, and any new table — not just 3 pages.
5. **Enterprise features on every table:** quick + advanced search; side filter + save/reuse filters; show/hide + reorder columns; asc/desc + multi-column sort; export Excel/PDF/(CSV optional); density Compact/Comfortable (persisted); persist page size + current page; multi-select + bulk actions; refresh without page reload; rich empty state (message + suggested action).
6. **Quality:** no workflow change, no breakage, all browsers, fast, desktop-first, scalable.
7. **Polish (from screenshots):** top KPI cards are large → make smaller/collapsible on operational pages; unify action-button order + icons across pages; unify status-badge sizes/style; unify financial-number alignment + fixed LYD decimal formatting.

**Your top 3:** ① Sidebar filters ② Unify all tables ③ Newest-first default.

## Architectural note (why sidebar filters is its own increment)

The shared `UnifiedMedicalTable` renders the **table body only**; **each page renders its own filters/search/action bar above it**. So "move filters to a sidebar" and "unify all filter UIs" require a reusable **EnterpriseTableShell** (sidebar panel + unified toolbar) that pages adopt one at a time. Building that shell + adopting the 4 heaviest pages is 2.1-B.2; the rest follow. This keeps each step small, reviewable, and reversible — exactly the lifecycle you mandated.
