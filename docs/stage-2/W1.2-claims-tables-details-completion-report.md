## Analyze

The Claims Operational Report renders through `pages/reports/claims/index.jsx` + `components/reports/claims/ClaimsTable.jsx`. Findings that shaped the work:
- The **claim number column displayed the internal `id`** (251, 151…); the official `claimNumber` ("CLM-251") sits in `_raw` (API confirmed to return it).
- **Number formatting was inconsistent**: the main table used a grouped LYD formatter, but the service-items table bypassed it with raw `.toFixed(2)` (no thousands separator). A canonical `formatCurrency` already exists in `utils/formatters` (duplicated locally in ClaimsTable).
- The **expanded service-items area was a dense nested table** with the key financials cramped (company/member stacked in one narrow column) and columns in database order.
- Tools (Export, Refresh, Filters) consumed vertical space above the table.

## Implement — the six improvements

**#1 — Enterprise Workspace Sidebar (reusable).** New `components/common/WorkspaceSidebar/` — a collapsible right-side drawer + a compact trigger (with an **active-filter badge**). All page tools now live inside it: **Refresh, Export (Excel), Print**, and the **Filters** (via a new `embedded` mode added to `EnterpriseFilters`, which renders fields + chips inline). The Claims page header is now slim (title + record count + the single **أدوات العمل** trigger), so **the table begins immediately below the header** — the toolbar no longer consumes vertical space. Reusable across Claims, Provider Portal, Settlements, Provider Contracts.

**#2 — Claim number as official reference.** The claim-number column now shows `claimNumber` ("CLM-251"), not the internal id. (Default sort remains id-desc = newest first, and CLM-N preserves that order visually.)

**#3 — Key financials immediately visible.** The expanded area now opens with a **summary strip** surfacing the claim's key figures at a glance: **إجمالي المطلوب · حصة الشركة (Company Liability) · تحمّل المريض (Member Co-payment) · المرفوض (Rejected) · المبلغ المعتمد (Approved)** — color-coded.

**#4 — Column order for operational review.** Line-item columns reordered to match the review flow: *service → quantity → unit price → total → coverage% → **company liability** → **member co-payment** → **rejected** → (benefit cap, remaining as muted context)*. Company and member shares are now **separate, clearly-visible columns** (previously cramped/stacked).

**#5 — Review-workspace feel.** The expanded area is now a bordered **review card** (header + financial summary strip + clean line table) rather than a raw nested table.

**#6 — Standardized LYD formatting.** Removed the local duplicate formatter; the whole page now uses the canonical `formatCurrency` from `utils/formatters` — consistent thousands separators + fixed 2 decimals everywhere (main columns show `د.ل`; dense cells use the same grouped 2-decimal number without the symbol).

**No data/query/workflow change:** filter changes still route to the page's existing setters; the claim/line data and permissions are unchanged.
