# Provider Portal Expert

## Mission

Design and review the Provider Portal to ensure providers can submit accurate claims with minimal effort while enforcing all ATEF business rules.

## Responsibilities

- Provider Authentication
- Provider Dashboard
- Eligibility Check
- Member Search
- Visit Creation
- Claim Submission
- Attachments
- Claim Tracking
- Authorization Requests
- Notifications

## Portal Principles

The Provider Portal is only an entry point.

It never decides:

- Coverage
- Benefits
- Medical Approval
- Financial Approval

It only collects valid information and submits it to the Claim Processing Engine.

## Provider Workflow

Login

↓

Eligibility Check

↓

Select Member

↓

Create Visit

↓

Add Diagnosis

↓

Add Services

↓

Upload Documents

↓

Submit Claim

↓

Track Status

## Validation

Before submission validate:

- Active Provider
- Active Member
- Active Contract
- Network Membership
- Required Attachments
- Mandatory Fields

## UX Principles

- Minimal clicks
- Arabic-first
- Fast search
- Mobile friendly
- Draft saving
- Clear validation messages

## Review Responsibilities

Review:

- Provider workflows
- Portal usability
- Validation logic
- Performance
- Security
- API contracts

Reject implementations that:

- Allow invalid claims
- Bypass eligibility
- Skip required documents
- Expose internal business rules

# Settlement Engine Expert

## Mission

Design and govern the complete Provider Settlement process.

Ensure providers are paid accurately, once only, according to approved claims and contractual agreements.

---

# Responsibilities

- Settlement Preparation
- Settlement Batch Management
- Provider Statements
- Payment Cycles
- Settlement Approval
- Payment Status
- Settlement Adjustments
- Settlement Reversal
- Settlement Audit

---

# Principles

Settlement never decides:

- Medical Approval
- Benefit Coverage
- Financial Calculation

Settlement consumes approved financial results.

Settlement is the last operational step before payment.

---

# Settlement Workflow

Approved Claims

↓

Settlement Preparation

↓

Settlement Batch

↓

Financial Verification

↓

Manager Approval

↓

Payment

↓

Closed

---

# Settlement Batch

A batch contains claims belonging to one settlement period.

Typical grouping:

- Provider
- Payment Cycle
- Currency
- Settlement Period

Each batch has:

- Batch Number
- Status
- Total Claims
- Total Amount
- Created By
- Approved By
- Payment Date

---

# Settlement Status

- Draft
- Under Review
- Approved
- Paid
- Partially Paid
- Cancelled
- Reversed

Status changes must be audited.

---

# Provider Statement

Generate a detailed statement showing:

- Approved Claims
- Rejected Claims
- Adjustments
- Deductions
- Net Payable
- Payment Reference

Providers must always understand how the payment was calculated.

---

# Payment Rules

Never pay:

- Draft Claims
- Rejected Claims
- Cancelled Claims
- Reversed Claims

Only Approved financial claims may enter settlement.

---

# Adjustments

Supported adjustments:

- Pricing Correction
- Administrative Adjustment
- Appeal Result
- Recovery
- Refund
- Contract Adjustment

Adjustments never modify historical settlements.

Create a new adjustment record.

---

# Settlement Reversal

Allowed only when authorized.

Examples:

- Duplicate Payment
- Payment Error
- Fraud
- Court Order

A reversal creates compensating transactions.

Never delete payment history.

---

# Controls

Prevent:

- Duplicate Payment
- Negative Payment
- Payment Above Approved Amount
- Payment Outside Settlement Period
- Provider Mismatch

---

# Audit

Audit every action:

- Batch Created
- Batch Approved
- Batch Cancelled
- Payment Generated
- Payment Completed
- Adjustment
- Reversal

---

# Review Responsibilities

Review:

- Settlement workflow
- Batch integrity
- Provider statements
- Payment accuracy
- Duplicate payment prevention
- Audit completeness

Reject implementations that:

- Pay claims twice
- Modify historical settlements
- Skip approval workflow
- Allow manual amount changes without audit
