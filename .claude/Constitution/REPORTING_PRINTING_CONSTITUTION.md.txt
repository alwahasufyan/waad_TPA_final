# Waad TPA Reporting & Printing Constitution

Version: 1.0

This document defines the mandatory standards for all reports, printed documents and PDF generation within the Waad TPA Platform.

Reports are business deliverables.

PDF files are official business documents.

Printing quality is part of product quality.

---

# Reporting Philosophy

Reports exist to support operational and management decisions.

Every report must answer a business question.

Never generate reports simply because data exists.

---

# Report Categories

Operational Reports

Financial Reports

Medical Reports

Settlement Reports

Management Reports

Audit Reports

Regulatory Reports

Provider Reports

Employer Reports

---

# Report Design

Every report should clearly answer:

What happened?

When?

Who?

Why?

How much?

What action is required?

Avoid unnecessary information.

Highlight important information.

---

# Report Layout

Every report should contain:

Company Logo

Report Title

Report Number (optional)

Generated By

Generated Date

Generated Time

Page Number

Applied Filters

Footer

Confidentiality Label (optional)

---

# Filtering

Every report must support filtering whenever applicable.

Common filters include:

Date Range

Employer

Provider

Member

Claim

Reviewer

Medical Category

Status

Benefit Policy

Settlement Cycle

Never generate huge reports without filters.

---

# Report Performance

Large reports must never block users.

Generate asynchronously when necessary.

Provide progress indicators.

Allow users to continue working while reports are generated.

---

# Report Accuracy

Every financial report must reconcile with the Financial Ledger.

Every medical report must reconcile with approved claims.

Totals must always match underlying data.

Never approximate financial values.

---

# Tables

Every report table should support:

Sorting

Grouping

Totals

Subtotals

Running Totals

Column Alignment

Multi-page Tables

Repeated Headers

---

# Financial Reports

Financial reports must display:

Requested Amount

Approved Amount

Rejected Amount

Discount

Patient Share

Insurance Share

Net Payable

Currency

Totals

Never hide calculation components.

Financial calculations must always be transparent.

---

# Medical Reports

Medical reports should include:

Diagnosis

Medical Category

Medical Service

Provider

Reviewer

Decision

Medical Notes

Authorization Status

Coverage Information

---

# Settlement Reports

Settlement reports should include:

Settlement Cycle

Provider

Claim Count

Gross Amount

Adjustments

Recoveries

Net Amount

Payment Status

Payment Date

Reference Number

---

# Audit Reports

Audit reports must include:

Timestamp

User

Action

Entity

Previous Value

New Value

Reason

IP Address (if available)

Correlation ID

Audit reports must never be editable.

---

# Export Formats

Every business report should support:

Print Preview

PDF

Excel

CSV (optional)

Do not support formats that lose business meaning.

---

# Printing

Printing is mandatory.

Every printable report should support:

A4

Portrait

Landscape (when required)

Automatic Page Breaks

Repeated Table Headers

Margins suitable for filing

Professional spacing

---

# PDF Standards

Every generated PDF must support:

RTL

Unicode Arabic Fonts

Arabic Text Shaping

Correct Line Wrapping

Automatic Pagination

Professional Typography

Embedded Fonts

Vector Logo

Searchable Text

Selectable Text

Archive Friendly

---

# Arabic Documents

Arabic documents are the primary output.

Support:

RTL

Arabic punctuation

Mixed Arabic/English text

Medical terminology

English codes

Arabic names

Proper alignment

---

# English Documents

English reports must preserve:

Medical codes

Financial precision

Consistent terminology

Professional formatting

---

# Currency

Default currency:

Libyan Dinar (LYD)

Example:

1250.500 د.ل

Display currency consistently.

Never mix currencies inside one report without clear indication.

---

# Dates

Display:

DD/MM/YYYY

Time:

24-hour format

Every report must show generation date and time.

---

# Printing Identity

Every official document should support:

Company Branding

Logo

Address

Contact Information

Document Number

Version

Generated By

Approval Area

Signature Area

Stamp Area

QR Code (future)

Barcode (future)

---

# Watermarks

Support:

Draft

Copy

Confidential

Cancelled

Paid

Approved

Rejected

Watermarks should be configurable.

---

# Digital Signature

Reports should be prepared for future digital signatures.

Do not hardcode signature images.

Signature areas must remain configurable.

---

# Archiving

Generated reports should remain reproducible.

A regenerated report using the same historical data should produce identical financial values.

Historical reports must never change because of future configuration changes.

---

# AI Reporting Behavior

Before creating or modifying any report Claude must verify:

Business purpose

Target audience

Required filters

Financial correctness

Medical correctness

Print quality

Arabic quality

PDF quality

Performance

If any requirement is missing,

request clarification before implementation.

---

# Final Principle

Every report represents the organization.

Every PDF may become a legal document.

Every printed page should be suitable for executive review, provider communication and regulatory submission without manual editing.
